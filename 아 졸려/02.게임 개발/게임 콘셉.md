# CLAUDE.md — 게임 제작 트랙

이 파일은 게임 프로젝트의 행동 지침서다. 프로젝트 폴더 맨 위(루트)에 두면 Claude가 작업할 때마다 먼저 읽고 따른다.

이 파일은 **고정된 규칙서가 아니다.** 아래 "내 게임 정의" 칸을 본인이 채우면, Claude가 그 게임에 맞춰 작업한다. 어디에 무엇을 쓰는지 안내를 따라 채운다.

---

## ■ 내 게임 정의 (여기를 채운다)

> 이 칸이 핵심이다. 채운 내용대로 게임이 만들어진다. 빈칸을 본인 게임으로 바꾼다.

- **게임 한 줄 정의**: (예: 떨어지는 장애물을 피하며 오래 버티는 게임)
  - →  미래에 있던 당신이 과거의 당신을 보기 위해서 달린다.. 그저 달린다..
  - 쓰는 법. "무엇을 하면 무엇이 되는가"를 한 문장으로. 이게 게임의 중심이다.

- **장르**: (예: 피하기 / 모으기 / 점프 / 클리커 / 퀴즈 / 슈팅)
  - → 원 버튼 게임, 러닝 액션

- **조작**: (예: 좌우 화살표로 이동, 스페이스로 점프)
  - →  스페이스바로 점프

- **목표 / 점수**: (예: 부딪히면 게임오버, 버틴 시간이 점수)
  - → 부딪히면 게임 오버, 버틴 시간이 점수

- **플랫폼**: (예: 웹 브라우저 HTML / 로블록스)
  - →  Unity3D로 제작 unity6.5(6000.5.1f1)버전

- **혼자 / 여럿**: (예: 혼자 / 여러 명이 같이)
  - → 솔로잉 게임
  - 쓰는 법. 여러 명이면 "각자 캐릭터가 따로 보이고 점수가 개별 집계"라고 적는다.

- **분위기 / 느낌**: (예: 밝고 경쾌한 / 어둡고 긴장감 있는)
  - → 미래에서 시작할때는 어둡고 두려운 분위기지만 과거의 나를 찾아가면서 밝아짐

> 다 못 정했어도 괜찮다. 정한 것만 적고 시작한 뒤, 하면서 채워 넣는다.

## ■ 작업 규칙 (내 게임에 맞게 고쳐도 된다)

> 아래는 권장 규칙이다. 내 게임에 안 맞으면 고치고, 필요한 규칙은 새로 추가한다. 규칙을 바꾸면 다음 작업부터 그대로 적용된다.

### 게임 전용

옛날에 컴투스에 있던 게임인  " 놈 " 게임을 만들라고 한다. 
이번에는 미래에서 과거로 가는 스토리에 게임을 만들라고 한다.
스테이지는 4 스테이지로 만들거고 미래 -> 현재 -> 과거 -> 보스로 만들거다.
혼자서 하는 게임인데 점수는 멀티로 된다. 
미래에서는 주인공이 늙었기 때문에 이동속도가 느리게 설정됨 : 기준점 1speed
현재는 미래보다 젊기 때문에 이동속도가 조금더 빠름 : 2speed
과거는 현재보다 더 젊기 때문에 이동속도가 더 빠르다 : 2.5speed
보스에서는 전투를 할 수 있게 한다.

- **배포까지가 완성.** "이 게임을 배포해서 접속 링크를 만들어줘. 안 되면 단계를 알려줘"로 마무리한다.

### 공통 (모든 작업)

- **생각부터, 그다음 코딩.** 멋대로 가정하지 않는다. 애매하면 멈추고 묻는다.
- **단순하게.** 시킨 것만 만든다. 길게 짠 코드가 짧아질 수 있으면 다시 짠다.
- **한글은 마침표로.** 한글 문장은 마침표(.)로 끝낸다. 콜론(:)으로 끝내지 않는다.
- **새 파일 첫 줄에 한글 주석.** 그 파일의 역할을 한 줄 한글로. 예: `-- 플레이어 이동과 점프를 처리하는 스크립트`
- **막히면 에러를 읽는다.** 추측 말고 실제 에러를 끝까지 읽는다. 에러를 그대로 복사해 Claude에게 보여주면 가장 빠르다.

## ■ 규칙을 추가·수정하는 법

- 작업하다 "이런 규칙이 있으면 좋겠다" 싶으면, 위 "작업 규칙"에 한 줄 추가한다.
- 예: "효과음은 무료 소스만 쓴다", "색은 3가지 안으로", "적은 항상 화면 위에서만 나온다".
- 내 게임에 맞지 않는 규칙은 지우거나 고친다. 이 파일은 내 게임에 맞춰 가는 문서다.
