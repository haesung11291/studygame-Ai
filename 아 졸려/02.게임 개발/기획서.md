# 게임 기획서 — 회귀 (回歸) *(가제)*

> 한 줄 정의. **미래의 내가 과거의 나를 보기 위해 달린다. 그저 달린다.**
> 컴투스 "놈"에서 영감을 받은 **원버튼 러닝 액션**.

---

## 1. 게임 개요

| 항목 | 내용 |
|------|------|
| 장르 | 원버튼 러닝 액션 (러너) |
| 플랫폼 | PC 웹브라우저 (Unity WebGL) |
| 엔진 | Unity 6.5 (6000.5.1f1), 2D |
| 플레이 인원 | 솔로 플레이 (점수는 순위로 경쟁) |
| 조작 | 스페이스바 하나 (점프) |
| 한 판 길이 | 스테이지당 약 20초, 전체 4스테이지 |

### 컨셉 / 스토리
늙은 미래의 주인공이 과거의 자신을 만나기 위해 시간을 거슬러 달린다. 미래에서 출발할 때는 **어둡고 두려운** 분위기지만, 현재를 지나 과거로 갈수록 주인공은 젊어지고 세계는 **점점 밝아진다.** 마지막에는 보스와의 결전이 기다린다.

---

## 2. 핵심 재미 (Core Loop)

1. 캐릭터가 자동으로 달린다.
2. 장애물이 다가오면 **스페이스바로 점프**해 피한다.
3. 부딪히면 게임 오버. **버틴 시간이 점수**가 된다.
4. 100점을 넘기면 다음 시대(스테이지)로 넘어간다.
5. 시대가 바뀔 때마다 속도·방향·분위기가 달라져 난이도가 오른다.

> 단순한 한 버튼 규칙 위에, **화면 회전**과 **방향 전환**이라는 "놈" 특유의 변주를 얹어 긴장감을 만든다.

---

## 3. 조작

| 입력 | 동작 |
|------|------|
| 스페이스바 | 점프 (바닥에 있을 때만) |
| "다시 시작" 버튼 클릭 / 엔터 키 | 게임 오버·승리 후 재시작 |

---

## 4. 게임 시스템

### 4.1 스테이지 구성 (4스테이지)

| 스테이지 | 시대 | 이동 속도 | 진행 방향 | 분위기 / 배경 |
|---------|------|:--------:|-----------|--------------|
| 1 | 미래 | 1.0 (기준) | 정방향 (→로 옴, 플레이어 왼쪽) | 어두운 하늘·달·별·네온 빌딩 |
| 2 | 현재 | 2.0 | 역방향 (←로 옴, 플레이어 오른쪽) | 밝은 하늘·해·구름·빌딩 |
| 3 | 과거 | 2.5 | **구간마다 역→정→역→정** (플레이어 가운데) | 파란 하늘·큰 해·언덕·나무 |
| 4 | 보스 | — | 러닝 없음 (전투) | 붉은 달·어두운 기둥·붉은 기운 |

- 주인공은 과거로 갈수록 젊어지므로 **이동 속도가 점점 빨라진다.**
- 과거 스테이지는 회전 지점마다 진행 방향이 뒤집혀 가장 혼란스럽다.

### 4.2 화면 회전 — "놈" 기믹
- 스테이지 점수가 **30 / 60 / 90점**이 될 때마다 화면이 플레이어를 중심으로 **90도씩 회전**한다.
- 물리(중력·점프·장애물)는 그대로 두고 시점만 돌려, 벽을 타고 달리는 듯한 감각을 준다.

### 4.3 방향 전환
- 스테이지별 진행 방향은 4구간(회전 지점 기준)으로 정의된다.
- 미래는 계속 정방향, 현재는 계속 역방향, **과거는 구간마다 역/정/역/정으로 교차**한다.
- 방향이 고정인 스테이지에서는 플레이어가 한쪽에, 방향이 바뀌는 스테이지에서는 가운데에 서서 양쪽 반응 거리를 공평하게 한다.
- 장애물은 각자 생성 시점의 방향을 기억해, 방향이 바뀌어도 지나간 장애물이 되돌아오지 않는다.

### 4.4 보스전 (스테이지 4)
- 달리기가 아닌 **전투**. 보스가 오른쪽에 버티고 서서 발사체를 쏜다.
- 플레이어는 왼쪽에서 **점프로 발사체를 회피**한다. 맞으면 게임 오버.
- 발사체를 **하나 피할 때마다 보스 체력이 1 감소**(총 20). 상단에 BOSS 체력바 표시.
- 보스 체력이 0이 되면 **승리**. 처음(미래)부터 다시 도전할 수 있다.

### 4.5 점수 & 순위
- 점수는 **초당 5점**씩 오른다(버틴 시간 기반).
- 점수는 스테이지를 넘어 **누적**된다(미래~보스 합산 총점).
- 게임 오버·승리 시 **순위표(TOP 5)** 와 내 점수, **이름 입력칸**을 보여준다.
- 기록은 기기(브라우저)에 저장되어 다시 켜도 유지된다. → 솔로 플레이지만 점수로 경쟁하는 "멀티 순위" 구현.

---

## 5. 아트 / 비주얼

- 외부 이미지 에셋 없이, **코드로 사각형 도형을 조합해** 캐릭터·배경·보스를 만든다(블록 아트 스타일).
- **캐릭터**: 몸·머리·팔·다리로 구성. 달릴 때 팔다리를 교차로 흔들고, 점프 중엔 도약 자세를 잡는다.
- **보스 몬스터**: 붉은 몸통에 뿔 2개, 번뜩이며 빛나는 눈, 팔. 천천히 들썩이는 애니메이션.
- **배경(맵)**: 스테이지마다 다른 테마로 시대와 분위기를 표현.
- **그리기 순서**: 배경 → 바닥 → 장애물/보스 → 캐릭터 순으로 겹쳐 깊이감을 준다.

---

## 6. UI / UX

- **HUD**: 좌상단 누적 점수, 보스전에서는 상단 BOSS 체력바.
- **게임 오버 / 승리 화면**: 내 점수, 순위 TOP 5(내 기록 강조), 이름 입력, **"다시 시작" 버튼**.
- 재시작은 버튼 클릭 또는 엔터 키로 한다(이름 입력과 겹치지 않게 점프의 스페이스와 분리).

---

## 7. 기술 사양

- Unity 6.5 (6000.5.1f1), 2D, Built-in 렌더 파이프라인.
- 씬은 코드(부트스트랩)로 구성. 실행 시 카메라·바닥·플레이어·배경·매니저를 자동 생성한다.
- 스테이지는 씬 이름으로 설정을 구분한다(`Main`=미래, `Stage2`=현재, `Stage3`=과거, `Stage4`=보스).
- 주요 스크립트: `GameManager`(전체 흐름), `PlayerController`(점프), `Obstacle`/`BossProjectile`(장애물·발사체), `RunAnimator`(캐릭터 애니메이션), `BossMonster`(보스), `Leaderboard`(순위 저장), `PrimitiveArt`(도형 아트).

---

## 8. 배포

- **WebGL**로 빌드해 웹브라우저에서 바로 플레이.
- 로컬/임시 링크로 테스트 후, 무료 호스팅(Netlify 등)에 올려 영구 공개 링크를 만든다.
- "배포까지가 완성" 원칙에 따라, 접속 링크 생성을 최종 목표로 한다.
