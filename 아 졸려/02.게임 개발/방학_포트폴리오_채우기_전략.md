# 방학 포트폴리오 채우기 전략
> 작성일: 2026-06-24 | 대상: 정회성 포트폴리오 100문 100답

---

## 현황 진단

| 구분 | 문항 수 |
|------|---------|
| 잘 채워진 항목 | 약 42개 |
| **"해당 없음" 또는 너무 짧은 항목** | **약 38개** |
| 기획자로서 영구 해당 없음 (백엔드/DB 등) | 약 20개 |
| **실제로 채워야 할 항목** | **약 18~20개** |

→ 핵심만 채우면 완성도가 크게 올라간다. 전부 채울 필요 없음.

## Phase 1 — 지금 당장 30분 안에 채울 수 있는 것

> 생각만 하면 바로 답할 수 있는 주관식 문항. 카페나 방에서 바로 작성.

| Q번호 | 질문 | 힌트 |
|-------|------|------|
| **Q20** | 학업 또는 교육 관련 수상 경력이 있나요? | Q98에서 "교내 장학생으로 선발"이라고 이미 언급함. 여기에 적으면 됨 |
| **Q67** | 본인의 약점은 무엇이며, 어떻게 개선하고 있나요? | 가장 솔직하게 1~2줄. (예: 개발 구현보다 기획에 집중되어 있어 코드 속도가 느린 편 → C++ 꾸준히 학습 중) |
| **Q72** | 실패 경험에서 배운 가장 큰 교훈은? | 낚시 소녀 에셋 기획 과정에서 시행착오가 있었을 것. 그걸 쓰면 됨 |
| **Q76** | 1년 후 나는 어떤 모습이고 싶나요? | 메이플 월드 대회 결과 반영 + 취업 준비 시작 |
| **Q77** | 3년 후 커리어 목표는? | 게임 회사 입사 후 시스템 기획 담당 → 리드 기획자 |
| **Q80** | 이상적인 회사 문화와 팀 분위기는? | 솔직하게. 수평적이고 일정 준수하는 팀, 기획 의도를 존중하는 문화 등 |
| **Q81** | 원하는 회사 규모는? | 스타트업 or 중견기업 (게임 콘텐츠 개발 가능한 규모면 충분) |
| **Q88** | 즐겨 참고하는 기술 블로그, 뉴스레터, 커뮤니티가 있나요? | 유튜브 채널명 + 게임 업데이트 분석하는 커뮤니티 (인벤, 블라인드 등) 적으면 됨 |
| **Q92** | 번아웃이나 슬럼프를 어떻게 극복하나요? | 게임 플레이로 리셋. 소드아트 온라인처럼 몰입할 콘텐츠 찾기 등 |
| **Q95** | 개발자가 되기로 결심한 계기는? | "No game, No Life" — 기계 수리에서 게임으로 커리어 전환한 이야기 이미 Q35에 있음. 그 내용 요약해서 쓰면 됨 |
| **Q99** | 지금 당장 배우고 싶은 기술이나 도전하고 싶은 것이 있다면? | 메이플 월드 Lua 스크립트 + 가챠 수치 밸런싱 심화 → 대회 준비와 연결 |

**소요 시간 예상: 30~60분**
**방법: 아래 프롬프트를 Claude에 복붙 후 초안 생성 → 본인 말투로 수정**

```
나는 게임 기획자 취준생이야. 아래 질문에 대해 자연스럽고 진솔하게 2~3문장으로 답변 초안을 작성해줘.
배경: 유한대학교 게임콘텐츠과 졸업, C++/C# 기초, Unity 엔진 경험, AI 툴 적극 활용
질문: [여기에 질문 붙여넣기]
```

## Phase 2 — 정보 수집 후 채울 것 (1~2일 내)

> 찾아보거나 확인해야 하는 것들. 오래 걸리지 않지만 자료가 필요함.

| Q번호 | 질문 | 할 일 |
|-------|------|-------|
| **Q3** | 연락 가능한 이메일 주소 | 취업용 이메일 하나 정하기 (Gmail 권장) |
| **Q5** | GitHub 프로필 링크 | GitHub URL 확인 후 기재 |
| **Q7** | Notion 블로그 URL | Notion 포트폴리오 페이지를 웹 공유 설정하고 URL 기재 |
| **Q55** | 낚시 소녀 개발 기간 | 기억 더듬어서 대략적 기간 기재 (예: 2025.xx ~ 2026.02, 약 x개월) |
| **Q56** | 낚시 소녀 팀 구성 | 혼자였는지, 협업이 있었는지 확인 후 기재 |
| **Q60** | 낚시 소녀 GitHub 링크 | 레포 있으면 URL, 없으면 "원스토어 출시" 명시 |

**소요 시간 예상: 1~2시간 (자료 찾는 시간 포함)**

## Phase 3 — 메이플 월드 대회 진행하면서 채울 것

> 대화기록에서 확인된 대회 마감: **2026년 9월 중순**.  
> 대회 준비를 하면서 자동으로 채워지는 항목들.

| Q번호 | 질문 | 채워지는 시점 |
|-------|------|--------------|
| **Q64** | 진행 중이거나 곧 시작할 프로젝트가 있나요? | 지금 바로 채울 수 있음 (대회 준비 중이라고 명시) |
| **Q55/56** | 프로젝트 기간 / 팀 구성 (Project 2) | 대회 완료 후 |
| **Q59** | Project 2 결과 (성과, 반응) | 대회 결과 발표 후 |
| **Q62** | Project 2 기술 스택 상세 | Lua 스크립트 + 메이플 월드 에디터 → 대회 진행하면서 정리 |
| **Q98** | 가장 보람을 느꼈던 순간 | 대회 완료 후 한 줄 추가 가능 |

### 대회 준비 중 병행할 것

대화기록에 "논의 중이던 질문 3가지"가 미정 상태임.  
이걸 결정하면 기획 방향이 잡히고 → **Q51~Q65 프로젝트 항목이 자연스럽게 채워진다.**

```
우선순위: 장르 분위기 → 주인공 설정 → 레퍼런스 정리 순으로 결정
```

## Phase 4 — 건너뛰어도 되는 항목 (기획자는 "해당 없음"이 자연스러움)

> 억지로 채우면 오히려 어색해지는 항목. 그냥 두거나 한 줄 이유만 명시.

| Q번호 | 이유 |
|-------|------|
| Q38~Q40 (프론트/백엔드/DB) | 기획자로서 해당 없음이 정상. 억지로 채우지 말 것 |
| Q42 (CI/CD) | 마찬가지 |
| Q16 (부트캠프) | 없으면 없는 것 |
| Q26, Q27 (인턴/프리랜서) | 없으면 없는 것 |
| Q29, Q30 (오픈소스/해커톤) | 방학 중 해커톤 참가 가능하면 채울 수 있음, 아니면 유지 |

## 전체 순서 요약

```
1일차 (30~60분)
  → Phase 1 항목 11개 Claude 활용해서 초안 작성 → 내 말투로 수정

2일차 (1~2시간)
  → GitHub URL, Notion 공유 링크 정리
  → 이메일 정하기
  → 낚시 소녀 기간/팀구성 메모

방학 내내 (대회 진행하면서)
  → 대화기록 미정 질문 3개 답하면서 기획 시작
  → 대회 종료 후 Q55/Q56/Q59/Q60 채우기

9월 이후
  → 완성된 포트폴리오로 portfolio.html 업데이트
```

## 빠르게 답변 퀄리티 높이는 팁

1. **"해당 없음" → "아직 경험 없으나 ~를 목표로 하고 있다"** 로 바꾸면 훨씬 좋아보임
2. **Q91 메이플 일퀘** → 귀엽지만 공식 포트폴리오에는 살짝 다듬기. "게임 플레이를 통해 기획 트렌드를 직접 체험하는 루틴을 유지하고 있습니다" 정도로
3. **짧은 답변 (Q93 페이커, Q94 기초부터)** → 왜 페이커인지, 왜 기초부터인지 1문장 추가
4. **중복 내용** → Q10과 Q100이 비슷함. Q10은 한 줄, Q100은 감사 인사 형식으로 차별화
